# OpenCowork 核心功能黑盒测试用例集

> 本文档覆盖工程核心功能的端到端验收测试，采用黑盒视角，适合手工验收或转自动化回归。
> 测试用例按功能模块分组，每条用例包含：目的、步骤、预期结果。

---

## 1. 启动与基础界面

### TC01 启动与基础界面可用性

- **目的**：验证应用能正常启动，主界面完整加载且各区域可操作。
- **前置条件**：应用已安装并可执行。
- **步骤**：
  1. 启动应用。
  2. 检查主界面是否完整加载（主面板、侧边栏、工具栏可见）。
  3. 切换主功能区域：任务、历史、模板、技能、结果面板。
- **预期结果**：
  - 应用启动成功，无崩溃或白屏。
  - 主界面可见且无明显布局错乱。
  - 各核心入口可点击并正常切换。

---

## 2. 任务执行与结果

### TC02 普通任务创建与执行

- **目的**：验证基础任务创建和执行链路完整可用。
- **前置条件**：系统正常运行，无阻塞性错误。
- **步骤**：
  1. 输入一个简单任务，例如"总结这段文本"。
  2. 提交执行。
  3. 等待任务完成。
- **预期结果**：
  - 任务能正常创建并开始执行。
  - 执行结束后能看到结构化结果。
  - 结果内容与任务目标一致。

### TC03 结果优先展示

- **目的**：验证执行结果可在结果视图中正确展示。
- **前置条件**：至少完成一个任务并产生结果。
- **步骤**：
  1. 执行一个会产生结果的任务。
  2. 打开结果面板。
  3. 打开历史记录中的同一任务。
- **预期结果**：
  - 结果面板显示完整结果内容。
  - 历史中能找到该任务。
  - 历史记录和结果内容一致。

### TC04 任务历史可追溯

- **目的**：验证任务完成后可在历史中追溯。
- **前置条件**：至少有一个已完成的任务。
- **步骤**：
  1. 完成一个任务。
  2. 退出并重新打开应用。
  3. 查看历史列表。
- **预期结果**：
  - 历史记录保留。
  - 能看到任务名称、状态、时间和结果摘要。
  - 可重新打开历史任务查看详情。

### TC23 失败结果与错误提示

- **目的**：验证任务失败时用户能获得明确提示且系统不崩溃。
- **前置条件**：系统正常运行。
- **步骤**：
  1. 提交一个故意失败的任务（例如无效指令）。
  2. 观察失败提示、日志和结果页。
- **预期结果**：
  - 失败原因清晰展示给用户。
  - 错误日志可见。
  - 系统不崩溃，仍可继续使用。

### TC24 重启后状态保持

- **目的**：验证应用重启后核心数据不会丢失。
- **前置条件**：已有任务历史、模板、技能等数据。
- **步骤**：
  1. 创建任务、保存模板、安装技能、执行一次桌面任务。
  2. 关闭应用并重新打开。
  3. 检查历史、模板、技能和最近结果。
- **预期结果**：
  - 必要数据保持。
  - 历史和配置可恢复。
  - 不出现数据损坏或乱码。

## 3. 模板与复用

### TC05 模板保存与复用

- **目的**：验证任务可保存为模板并再次使用。
- **前置条件**：至少有一个成功完成的任务。
- **步骤**：
  1. 将该任务保存为模板。
  2. 在模板列表中找到它。
  3. 使用模板再次运行。
- **预期结果**：
  - 模板保存成功。
  - 模板列表中可见。
  - 复用后任务能正常执行。

### TC06 模板筛选与安装状态

- **目的**：验证模板库的筛选和安装状态展示正确。
- **前置条件**：模板列表中有已安装和未安装模板。
- **步骤**：
  1. 打开模板面板。
  2. 查看已安装模板与可安装模板。
  3. 切换筛选条件（全部 / 已安装 / 可安装）。
- **预期结果**：
  - 模板可按状态筛选。
  - 安装状态显示正确。
  - 已安装模板与未安装模板区分明确。

## 4. 技能生态

### TC07 技能库浏览与安装

- **目的**：验证技能发现、安装和启用流程完整可用。
- **前置条件**：技能列表可访问，网络正常。
- **步骤**：
  1. 打开技能面板。
  2. 浏览技能列表。
  3. 安装一个技能。
  4. 返回技能列表查看状态。
- **预期结果**：
  - 技能列表正常显示。
  - 安装过程有进度或成功反馈。
  - 安装后状态变为已安装或可用。

### TC08 技能更新与筛选

- **目的**：验证技能更新提示和更新筛选功能正常。
- **前置条件**：存在可更新的技能。
- **步骤**：
  1. 打开技能面板。
  2. 切换到"仅看可更新"筛选条件。
  3. 对可更新技能执行更新操作。
- **预期结果**：
  - 可更新技能被优先显示或筛出。
  - 更新后状态变化正确。
  - 更新结果有明确反馈。

## 5. 调度与 IM 集成

### TC09 定时任务执行

- **目的**：验证调度器能按计划触发任务。
- **前置条件**：调度功能已启用，任务未超时。
- **步骤**：
  1. 创建一个定时任务，设置短时间后触发。
  2. 等待触发并观察执行结果。
- **预期结果**：
  - 定时任务按时触发。
  - 任务执行状态正确更新。
  - 结果可在任务历史中查看。

### TC10 IM 消息接入

- **目的**：验证 IM 通道可接收消息并转成任务。
- **前置条件**：IM 集成已配置并在线。
- **步骤**：
  1. 从 IM 发送一条任务消息。
  2. 观察应用是否创建任务。
  3. 检查任务内容是否与消息一致。
- **预期结果**：
  - IM 消息能转成任务。
  - 任务内容正确。
  - 任务状态可追踪。

### TC11 IM 附件与结果回传

- **目的**：验证 IM 文件/图片附件处理和结果回传正常。
- **前置条件**：IM 集成已配置，支持文件收发。
- **步骤**：
  1. 通过 IM 发送带附件消息（图片或文件）。
  2. 观察是否创建任务并处理附件。
  3. 等待任务完成。
  4. 检查结果是否通过 IM 返回。
- **预期结果**：
  - 附件能进入任务处理链路。
  - 任务结果能回传到原会话。
  - 结果格式正确，无丢失。

## 6. MCP 工具支持

### TC12 MCP 工具能力可用

- **目的**：验证 MCP 客户端/服务器相关功能可用。
- **前置条件**：MCP 配置已就绪，至少有一个 MCP 工具可用。
- **步骤**：
  1. 打开 MCP 相关配置或工具入口。
  2. 查看可用工具列表。
  3. 选择一个工具执行调用。
- **预期结果**：
  - 工具可见且可调用。
  - 调用后有结果返回。
  - 工具调用不影响主任务流程。

---

## 7. 浏览器与视觉执行

### TC13 浏览器任务执行

- **目的**：验证浏览器目标的基础视觉/自动化任务可执行。
- **前置条件**：浏览器已启动并可访问目标页面。
- **步骤**：
  1. 选择浏览器执行目标。
  2. 输入一个网页任务，例如搜索、点击、输入。
  3. 执行任务。
- **预期结果**：
  - 任务能在浏览器中执行。
  - 页面状态按预期变化。
  - 结果可在结果面板查看。

### TC14 浏览器任务审批拦截

- **目的**：验证高风险浏览器/视觉动作会触发审批。
- **前置条件**：审批功能已开启。
- **步骤**：
  1. 提交一个包含高风险动作的任务，例如提交表单、发送数据、删除操作。
  2. 观察是否出现审批弹窗。
  3. 检查弹窗中显示的审批信息。
- **预期结果**：
  - 系统阻止直接执行。
  - 弹出审批窗口。
  - 显示风险原因、任务描述和动作列表。

### TC15 审批继续执行

- **目的**：验证审批通过后任务可继续执行并完成。
- **前置条件**：存在一个被审批拦截的任务。
- **步骤**：
  1. 触发审批弹窗（参考 TC14）。
  2. 点击"继续审批"或"批准并继续"。
  3. 观察任务是否继续执行到结束。
- **预期结果**：
  - 任务恢复执行。
  - 结果正确返回。
  - 审批状态更新为已通过。

## 8. 桌面执行路径

### TC16 VM 桌面执行主路径

- **目的**：验证 VM 作为主桌面路径可正常工作。
- **前置条件**：VM 桌面目标已配置，视觉任务功能可用。
- **步骤**：
  1. 选择桌面执行目标为 VM。
  2. 提交一个桌面任务，例如打开应用、编辑文件、保存。
  3. 观察执行过程与结果。
- **预期结果**：
  - 任务上下文包含桌面执行目标。
  - VM 允许的桌面动作可执行（打开应用、文件操作等）。
  - 最终结果能完成或给出可理解反馈。

### TC17 桌面审批弹窗展示执行后端

- **目的**：验证桌面审批能展示执行目标上下文（desktop / vm）。
- **前置条件**：VM 或桌面目标已配置，审批已开启。
- **步骤**：
  1. 在 VM 或桌面目标上执行高风险动作（例如打开文件、窗口聚焦）。
  2. 触发审批弹窗。
  3. 查看审批弹窗信息内容。
- **预期结果**：
  - 弹窗显示执行后端信息，例如 `desktop / vm`。
  - 显示风险原因和意图关键词。
  - 用户能据此判断是否继续。

### TC18 桌面恢复能力

- **目的**：验证桌面流程遇到窗口失焦、文件对话框、输入失败时可自动恢复。
- **前置条件**：VM 或桌面目标已配置，恢复机制已实现。
- **步骤**：
  1. 执行一个桌面任务。
  2. 人为制造窗口失焦、文件对话框未出现、输入未生效等场景。
  3. 观察系统是否自动恢复。
- **预期结果**：
  - 系统能给出恢复策略（日志或提示可见）。
  - 任务不会直接卡死。
  - 能重新观察并继续执行。

### TC19 容器桌面执行

- **目的**：验证容器桌面路径可运行。
- **前置条件**：容器桌面目标已配置。
- **步骤**：
  1. 选择桌面执行目标为容器（container）。
  2. 执行一个桌面任务。
  3. 观察执行与恢复行为。
- **预期结果**：
  - 容器目标能被正确识别。
  - 任务可执行。
  - 桌面上下文信息正确展示。

### TC20 原生桥接桌面执行

- **目的**：验证 native-bridge 路径可运行并保留高风险审批。
- **前置条件**：native-bridge 桌面目标已配置，审批已开启。
- **步骤**：
  1. 选择 native-bridge 桌面目标。
  2. 执行包含打开文件、窗口聚焦或下载等动作的任务。
  3. 观察审批和执行结果。
- **预期结果**：
  - 执行目标正确显示为 `desktop / native-bridge`。
  - 高风险动作会触发审批。
  - 审批通过后能继续执行。

## 9. 结果一致性与展示

### TC21 结果、运行记录与历史一致性

- **目的**：验证同一任务在结果页、运行页、历史页中的信息一致。
- **前置条件**：至少有一个已完成的任务。
- **步骤**：
  1. 完成一个任务。
  2. 分别打开结果面板、运行详情、历史记录。
  3. 对比摘要、状态、执行目标等信息。
- **预期结果**：
  - 三处信息一致，无矛盾。
  - 执行目标、结果摘要、时间线不冲突。
  - 展示内容可追溯。

## 10. 基准测试与回归

### TC22 基准任务与套件运行

- **目的**：验证 benchmark 任务和套件能正常运行。
- **前置条件**：P4 desktop smoke suite 已加载，至少一个 benchmark 可执行。
- **步骤**：
  1. 选择 P4 desktop smoke suite。
  2. 执行套件。
  3. 查看每个 benchmark 的结果和总体统计。
- **预期结果**：
  - 套件可启动。
  - 每个 benchmark 有独立结果。
  - 总体通过/失败统计正确。

### TC25 常规回归 smoke

- **目的**：覆盖最小发布回归集合，确保四条主链路同时可用。
- **前置条件**：系统正常运行，所有核心功能已实现。
- **步骤**：
  1. 运行一个浏览器任务。
  2. 运行一个桌面 VM 任务。
  3. 运行一个带审批的高风险任务。
  4. 运行一个 IM 入口任务（如果有 IM 集成）。
- **预期结果**：
  - 四条主链路都可工作。
  - 没有互相影响或阻断性错误。
  - 发布前没有严重阻碍。
